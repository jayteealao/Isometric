## [Unreleased]
### Breaking Changes
- `Shape()`, `Group()`, and related composables: parameter `color: IsoColor` renamed to `material: MaterialData`. This is a binary-incompatible change ã compiled callers using named-argument syntax (`color = ...`) must recompile and update call sites to `material = ...`. `IsoColor` still satisfies the new parameter as-is since it implements `MaterialData`.
- **Per-face DSL rename (`isometric-shader`).** The top-level `perFace { }` DSL function is renamed to `prismPerFace { }`; its builder scope class `PerFaceMaterialScope` is renamed to `PrismPerFaceMaterialScope`. Rationale: per-shape DSLs now exist for Pyramid (`pyramidPerFace`) and Octahedron (`octahedronPerFace`) ã the unprefixed name made Prism look shape-agnostic when it was not. Migration: find-and-replace `perFace {` ã `prismPerFace {` and `PerFaceMaterialScope` ã `PrismPerFaceMaterialScope` in call sites.
- **`Pyramid.paths.size` 4 ã 5.** `Pyramid` now emits five paths ã four triangular lateral faces at indices `0..3` and a rectangular base quad at index `4`. Downstream code that hard-coded `4` for pyramid iteration must either use `paths.size` or accept the new base face. See `Pyramid` KDoc for the full face layout.
- **`PerFace` + `IsoColor` sub-material Canvas rendering corrected.** Before this release, `prismPerFace { front = IsoColor.RED }` (or any `PerFace` with a non-`Textured` sub-material) silently painted the `PerFace.default` color instead of the per-face assignment on the Canvas backend. The fragment shader path (WebGPU) had the same silent-fallback bug. Both are now corrected: per-face `IsoColor` and per-face `Textured.tint` both reach the rasterizer. If you worked around the old behaviour by matching the PerFace default to the "intended" color, faces will now paint the actual per-face value.

### Features
- **`octahedronPerFace { }` DSL (`isometric-shader`).** New top-level DSL for building `IsometricMaterial.PerFace.Octahedron` with enum-keyed slot assignments. `face(OctahedronFace.UPPER_0, material)` replaces the `PerFace.Octahedron(byIndex = mapOf(OctahedronFace.UPPER_0 to material, ...))` boilerplate; `allFaces(material)` paints every face with one call. The raw `PerFace.Octahedron` constructor remains available as an escape hatch.
