Dominant cost: Prepare

Average prepare time across NONE rows: 36.64ms. At N=200, prepare alone exceeds 100ms. This is the biggest lever for any optimization.

Secondary cost: Hit-test

Under continuous interaction, hit-test averages 38.35ms. At N=200, hit-test reaches 106ms — comparable to prepare cost.

Not the bottleneck: Draw

Average draw time across all rows: 1.19ms. Draw is consistently cheap. Branches targeting draw-time (path caching, native canvas) face a narrower ceiling.

Branch: Prepared Scene Cache

Skips engine.prepare() on frames where the node tree, viewport, render options, and lighting are unchanged. Only enablePreparedSceneCache=true; all other flags off.

Methodological caveat

The current matrix uses non-zero mutation rates (0.10 and 0.50), which trigger cache invalidation. This proves the cache helps even under real invalidation pressure but does not replace the planned mutationRate=0.0 stable-scene probe for clean cache-reuse proof.

Methodology and Artifacts

How results were produced and where the raw data lives.

Comparison design

  • Baseline: all optimization flags false
  • Branch: only the single target flag set true
  • Same physical device (SM-F956B), same Android 34, same harness, same 24-scenario matrix
  • Self-tests validated in both runs
  • Matrix: 4 scene sizes × 2 mutation rates × 3 interaction patterns

Reading guide

  • avgFrameMs — primary decision metric (product-visible)
  • avgPrepareMs — subsystem diagnosis for scene prep
  • avgHitTestMs — subsystem diagnosis for interaction
  • allocatedMB / gcInvocations — cost-shifting indicators
  • p95 / p99 — tail latency when jank matters
