## 2026-04-26T19:20:32Z — shape — Round 4 (what can go wrong)

- **Threshold semantics for `countCloserThan`**: Return 1 when **strict majority** `result > points.size / 2`. Robust against single noisy vertex; matches original "most vertices on observer side" intent more faithfully than integer division.
- **Epsilon at Path.kt:128**: Loosen from `1e-9` to **`1e-6`** as an absolute threshold — sub-agent 2's analysis confirmed `1e-9` is too tight for the project's typical 0–100 coordinate range.
- **Defensive guards**: Add a debug-only invariant check that `closerThan(A,B,obs) == -closerThan(B,A,obs)` for non-coplanar pairs. Costs nothing in release.
- **Telemetry**: None. `isometric-core` is logger-free; preserve that contract. The new behaviour is correct, not exceptional.
